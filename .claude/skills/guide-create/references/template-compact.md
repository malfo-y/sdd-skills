# Guide Template (Compact)

> Canonical generation template for SDD guide documents. A guide is a feature-specific deep-dive and companion surface. It does not mirror the thin global spec outline. Instead, it expands feature context, usage flow, review focus, and verification notes in a bounded way.

## Section Mapping

| Guide Section | Source Idea | Purpose |
|---------------|-------------|---------|
| §1 Overview | 문제/배경/범위 | 기능 맥락 빠른 정렬 |
| §2 Core Design | 핵심 설계와 주요 결정 | 기능 단위 설계 판단 설명 |
| §3 Usage Scenario Guide | feature-level usage / expected result | 시나리오와 기대 결과 확장 |
| §4 Verification Notes | temporary validation / review focus | 검증 포인트 정리 |
| Appendix | 관련 코드 레퍼런스 | 선택적 탐색 힌트 |
