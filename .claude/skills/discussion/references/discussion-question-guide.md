# Discussion Question Guide

토론 스킬에서 사용하는 질문 전략 및 템플릿 가이드.

## 토론 단계별 질문 전략

### Phase 1: 탐색 (Exploration) - Round 1-3

목적: 토픽의 범위와 맥락을 이해한다.

| 질문 유형 | 템플릿 | 사용 시점 |
|-----------|--------|----------|
| 동기 확인 | "이 [토픽]을 고려하게 된 계기가 무엇인가요?" | Round 1 |
| 현황 파악 | "현재 [관련 영역]은 어떤 상태인가요?" | Round 1-2 |
| 목표 정의 | "이상적인 결과는 어떤 모습인가요?" | Round 2-3 |
| 제약 조건 | "꼭 지켜야 할 제약 조건이 있나요?" | Round 2-3 |

### Phase 2: 분석 (Analysis) - Round 4-6

목적: 대안을 비교하고 트레이드오프를 분석한다.

| 질문 유형 | 템플릿 | 사용 시점 |
|-----------|--------|----------|
| 대안 비교 | "A와 B 중 [기준]에서 어떤 것이 더 적합할까요?" | Round 4-5 |
| 트레이드오프 | "[장점]을 위해 [단점]을 감수할 수 있나요?" | Round 4-5 |
| 우선순위 | "가장 중요한 기준은 무엇인가요?" | Round 5-6 |
| 반론 제시 | "반대로 [대안]을 선택한다면 어떤 이점이 있을까요?" | Round 5-6 |

### Phase 3: 수렴 (Convergence) - Round 7+

목적: 결론을 도출하고 합의를 형성한다.

| 질문 유형 | 템플릿 | 사용 시점 |
|-----------|--------|----------|
| 결론 확인 | "지금까지 논의를 바탕으로 [결론]이 맞을까요?" | Round 7+ |
| 위험 확인 | "이 결정의 가장 큰 위험은 무엇이라고 보시나요?" | Round 7+ |
| 실행 확인 | "다음 단계로 어떤 행동이 필요할까요?" | Round 8+ |
| 최종 합의 | "이 결정으로 진행해도 괜찮으시겠어요?" | 마지막 라운드 |

## AskUserQuestion 옵션 설계 원칙

### 옵션 수
- **최소 2개, 최대 3개** + "토론 종료" 옵션 (총 최대 4개)

### 옵션 설계 기준
1. **상호 배타적**: 각 옵션이 겹치지 않아야 한다
2. **포괄적**: 주요 가능성을 모두 커버해야 한다
3. **명확**: 선택의 의미가 분명해야 한다
4. **균형**: 특정 방향으로 유도하지 않아야 한다

### "Other" 활용
- 카테고리 선택 시: "Other (직접 입력)" 포함
- 의견 질문 시: 포함하지 않음 (자유 응답은 대화로 수용)

## 리서치 트리거 조건

### 자동 리서치 (Step 2)
- 코드베이스 관련 토픽 → Explore agent 자동 실행
- 기술 비교 토픽 → general-purpose agent 자동 실행

### 수동 리서치 (Step 3 mid-discussion)
- 사용자가 "잘 모르겠어요 (추가 리서치 필요)" 선택
- 사용자의 주장에 대한 사실 확인 필요
- 특정 코드/파일에 대한 확인 필요
- 비교 자료가 필요한 트레이드오프 논의

### 리서치 스킵 조건
- 동일 주제에 대해 이미 리서치를 수행한 경우
- 토론이 수렴 단계(Phase 3)에 진입한 경우
- 사용자가 빠른 토론을 원하는 경우

## 토론 상태 추적 형식

### 내부 상태 (암묵적 관리)

```
## Discussion State
- Topic: [토픽]
- Current Phase: Exploration / Analysis / Convergence
- Round: N / MAX(10)

### Key Points
1. [논점]: [요약] (Round N)

### Decisions
1. [결정]: [근거] (Round N)

### Open Questions
1. [질문]: [맥락]

### Action Items
1. [항목]: [우선순위]

### Research Log
1. [주제]: [결과 요약] (Round N, agent type)
```
