# Discussion Question Guide

토론 스킬에서 사용하는 질문 전략 및 템플릿 가이드.

## 토론 단계별 질문 전략

페이즈 전환은 라운드 번호가 아닌 **커버리지 달성도**에 따라 결정된다. 아래 라운드 번호는 참고 예시일 뿐이다.

### Phase 1: 탐색 (Exploration)

목적: 토픽의 범위와 맥락을 이해한다.

**커버리지 항목:** `motivation`, `current_state`, `constraints`, `ideal_outcome`

| 질문 유형 | 템플릿 | 커버리지 |
|-----------|--------|----------|
| 동기 확인 | "이 [토픽]을 고려하게 된 계기가 무엇인가요?" | motivation |
| 현황 파악 | "현재 [관련 영역]은 어떤 상태인가요?" | current_state |
| 목표 정의 | "이상적인 결과는 어떤 모습인가요?" | ideal_outcome |
| 제약 조건 | "꼭 지켜야 할 제약 조건이 있나요?" | constraints |

### Phase 2: 분석 (Analysis)

목적: 대안을 비교하고 트레이드오프를 분석한다. **비판적 개입**으로 약점을 검증한다.

**커버리지 항목:** `alternatives`, `tradeoffs`, `priorities`, `critical_review`

| 질문 유형 | 템플릿 | 커버리지 |
|-----------|--------|----------|
| 대안 비교 | "A와 B 중 [기준]에서 어떤 것이 더 적합할까요?" | alternatives |
| 트레이드오프 | "[장점]을 위해 [단점]을 감수할 수 있나요?" | tradeoffs |
| 우선순위 | "가장 중요한 기준은 무엇인가요?" | priorities |
| 반론 제시 | "반대로 [대안]을 선택한다면 어떤 이점이 있을까요?" | alternatives |

**설계/아키텍처 토픽용 품질 렌즈 (Design Quality Lens):**

토픽 유형이 아키텍처/설계일 때 analysis 페이즈에서 아래 렌즈 질문을 최소 1회 다룬다. 좋은 설계의 기준을 프로빙 질문 형태로 제공한다.

| 렌즈 | 질문 템플릿 | 목적 |
|------|------------|------|
| 단일 책임 | "각 단위가 하나의 명확한 목적을 갖나요? 둘 이상의 책임이 섞였다면 어디서 나뉠까요?" | 경계 설계 |
| 인터페이스 | "이 단위를 사용하는 쪽은 내부 구현을 몰라도 쓸 수 있나요?" | 캡슐화 |
| 의존성 | "이 단위가 의존하는 것은 무엇이고, 그 의존은 필수인가요?" | 결합도 |
| 독립 이해/테스트 | "이 단위를 다른 부분과 분리해서 이해하고 테스트할 수 있나요?" | 격리성 |
| 변경 영향 | "내부를 바꿨을 때 소비자가 깨지지 않나요?" | 인터페이스 안정성 |
| 파일 크기 신호 | "어느 한 파일/단위가 비대해지고 있다면, 그것이 책임 과다 신호는 아닌가요?" | 분해 트리거 |

이 렌즈는 **필수 커버리지가 아니라 analysis 페이즈에서 설계 토픽에 대한 보조 커버리지**다. 설계 외 토픽(기술 선택, 리서치 등)에서는 생략 가능.

**비판적 개입 (critical_review):**

분석 단계에서는 최소 1회 비판적 질문으로 약점이나 숨은 가정을 검증한다.

- 사용자가 특정 방향에 합의/선호를 표현하면 그 방향을 직접 검증한다.
- 사용자가 중립 비교만 유지하면 가장 가능성이 높은 방향 또는 공통 가정을 임시 가설로 잡고 검증한다.
- 임시 가설도 어렵다면 "가장 우려되는 실패 시나리오" 또는 "가정이 틀리면 무엇이 깨지는지"를 묻는 질문으로 `critical_review`를 완료한다.

| 유형 | 템플릿 |
|------|--------|
| 약점 지적 | "이 방식이면 [엣지 케이스]에서 문제가 될 수 있는데, 어떻게 생각하세요?" |
| 반례 제시 | "비슷한 접근을 한 [사례]에서는 [문제]가 있었어요" |
| 숨은 가정 질문 | "이건 [가정]이 전제인데, 그게 깨지면요?" |
| 대안 제안 | "다른 관점에서 보면 [대안]도 가능한데, 고려해 보셨나요?" |
| 스케일 도전 | "지금은 괜찮지만 10배 커지면 이 구조가 버틸까요?" |
| YAGNI / 범위 축소 | "이 기능이 지금 꼭 필요한가요? 없으면 무엇이 깨지나요?" |

**강도 조절:** 연속 2라운드 비판 금지. 비판 후 대안/보완책 함께 제시. 사용자가 반박한 약점 재제기 금지. 비판은 질문 형태로.

### Phase 3: 수렴 (Convergence)

목적: 결론을 도출하고 합의를 형성한다. exploration + analysis 커버리지가 완료되면 진입.

**커버리지 항목:** `conclusion`, `risks`, `action_items`

| 질문 유형 | 템플릿 | 커버리지 |
|-----------|--------|----------|
| 결론 확인 | "지금까지 논의를 바탕으로 [결론]으로 정리할까요?" | conclusion |
| 위험 확인 | "이 결정의 가장 큰 위험은 무엇이라고 보시나요?" | risks |
| 실행 확인 | "다음 단계로 어떤 행동이 필요할까요?" | action_items |
| 최종 합의 | "이 방향으로 정리해도 괜찮을까요?" | conclusion |

## AskUserQuestion 옵션 설계 원칙

### 옵션 수
- **최소 2개, 최대 3개** + "토론 종료" 옵션 (총 최대 4개)

### 옵션 설계 기준
1. **상호 배타적**: 각 옵션이 겹치지 않아야 한다
2. **포괄적**: 주요 가능성을 모두 커버해야 한다
3. **명확**: 선택의 의미가 분명해야 한다
4. **균형**: 특정 방향으로 유도하지 않아야 한다

### "Other" 활용
- 카테고리 선택 시: "Other (직접 입력)" 포함
- 의견 질문 시: 포함하지 않음 (자유 응답은 대화로 수용)

## 리서치 트리거 조건

### 자동 리서치 (Step 2)
- 코드베이스 관련 토픽 → Explore agent 자동 실행
- 기술 비교 토픽 → general-purpose agent 자동 실행

### 수동 리서치 (Step 3 mid-discussion)
- 사용자가 "잘 모르겠어요 (추가 리서치 필요)" 선택
- 사용자의 주장에 대한 사실 확인 필요
- 특정 코드/파일에 대한 확인 필요
- 비교 자료가 필요한 트레이드오프 논의

### 리서치 스킵 조건
- 동일 주제에 대해 이미 리서치를 수행한 경우
- 토론이 수렴 단계(Phase 3)에 진입한 경우
- 사용자가 빠른 토론을 원하는 경우

## Stagnation Handling

라운드 수 상한은 두지 않는다. 대신 아래 정체 신호를 감지해 종료 fallback을 건다.

### 정체 신호
- 연속 라운드에서 `key_points` 또는 `decisions` 증가 없음
- 같은 open question이 반복되지만 진전 없음
- 사용자가 반복적으로 유보/모호 응답만 제공

### fallback 질문 패턴

정체 신호가 2회 연속이면 질문 범위를 아래 둘 중 하나로 줄인다.

```text
질문: "새 정보가 크게 늘지 않고 있습니다. 어떻게 진행할까요?"
1. "지금까지 정리"
2. "미결 1개만 더 논의"
3. "토론 종료 / 정리해줘"
```

사용자가 `미결 1개만 더 논의`를 택했는데 다음 라운드도 정체면, 남은 항목은 open question으로 기록하고 요약 단계로 이동한다.

## 토론 상태 추적 형식

### 내부 상태 (암묵적 관리)

```
## Discussion State
- Topic: [토픽]
- Current Phase: Exploration / Analysis / Convergence (커버리지 기반 판정)
- Round: N

### Coverage
exploration:
  motivation: ✅/❌  current_state: ✅/❌  constraints: ✅/❌  ideal_outcome: ✅/❌
analysis:
  alternatives: ✅/❌  tradeoffs: ✅/❌  priorities: ✅/❌  critical_review: ✅/❌
convergence:
  conclusion: ✅/❌  risks: ✅/❌  action_items: ✅/❌

### Key Points
1. [논점]: [요약] (Round N)

### Decisions
1. [결정]: [근거] (Round N)

### Open Questions
1. [질문]: [맥락]

### Action Items
1. [항목]: [우선순위]

### Research Log
1. [주제]: [결과 요약] (Round N, agent type)
```
