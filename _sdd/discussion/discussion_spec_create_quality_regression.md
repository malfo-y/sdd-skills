# 토론 요약: 전체 SDD 스킬 품질 회귀 분석 및 복원 전략

**날짜**: 2026-03-09
**라운드 수**: 8 (2회 세션)
**참여 방식**: 구조화된 토론 (discussion skill)

## 핵심 논점 (Key Discussion Points)

1. **이전 스킬(24618c3)은 이해 가능한 스펙을 생성**: 구체적 예시, 풍부한 서술, 설계 결정 테이블, 코드 스키마 예시가 포함되어 사람이 프로젝트를 이해할 수 있었음
2. **현재 스킬은 지나치게 간략한 스펙만 생성**: exploration-first 전환(커밋 234c4f3) 과정에서 예시가 placeholder로 대체되어 에이전트가 얕고 추상적인 스펙만 생산
3. **근본 원인은 예시 기반 학습(example-driven learning)의 상실**: 템플릿은 "무엇을 쓰라"고 말하지만, "어떻게 잘 쓰는지"의 구체적 모범 사례가 사라짐
4. **문제는 spec-create에 국한되지 않음**: 14개 전체 스킬이 동일한 exploration-first 전환으로 이해 깊이를 상실. 공통적으로 설명 깊이, 예시 풍부함, 절차적 가이드가 축소됨

### 공통 손실 패턴

1. **예시 축소**: 모든 스킬에서 구체적 예시가 placeholder나 메타 가이드로 대체
2. **워크플로우 단순화**: Large/Medium/Small 분기 → 단일 4단계 파이프라인
3. **설명 깊이 감소**: 절차적 설명과 before/after 비교가 축소
4. **공통 매니페스토 삽입**: 모든 스킬에 동일한 탐색형 매니페스토가 추가되어 개별 스킬의 맥락적 설명 공간 압축
