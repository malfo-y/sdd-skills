## 토론 배경 및 초기 콘텍스트 (Background / Initial Context)

- **사용자 문제 제기**: discussion 스킬의 진행 방식과 결과 요약 문서가 `docs/software_3_loop_engineering.md`의 Software 3.0 / Loop Engineering 철학과 align되어 있길 원함.
- **토론을 시작한 배경**: 같은 날 완료된 전체 conformance audit(31 유닛 CLEAR + 블라인드 판별력 테스트 통과)의 후속. audit은 규범 *최소 부합*을 판정했고, 이번 토론은 그보다 깊은 층위 — 철학의 모범적 체화 — 를 다룸.
- **현재 상태**: `.claude/skills/discussion/SKILL.md`(408줄)는 당일 audit에서 **CLEAR** (PASS 6 / N/A 5 / FAIL 0, `docs/audits/discussion_conformance.md`). 진행 루프(수렴 신호·stagnation fallback·비판적 개입·soft deferral)는 loop 구조가 이미 잘 체화됨.
- **범위와 제외 범위**: 범위 = discussion 스킬의 진행 방식 + 요약 문서 형식의 철학 정렬 방안 도출. 제외 = 실제 SKILL.md 수정 실행(후속 작업), 타 스킬로의 확대, loop-eng 용어의 표면 도입.
- **수집한 근거**: `docs/software_3_loop_engineering.md` (§3.1 자연어=실행 인터페이스, §4 루프 구조, §6 좋은 Loop 5조건, §10 실패 모드), `docs/audits/discussion_conformance.md`, 당일 세션의 규범 원칙(스킬은 부합만, 참조 안 함 / 어휘 1:1 고정표).

## 핵심 논점 (Key Discussion Points)

1. **정렬 갭의 위치**: 진행 루프는 이미 강함(§6.1/6.3/6.5, §10 대응 모두 체화 — audit CLEAR가 방증). 진짜 갭은 산출물 — (갭 1) 요약이 "회의록"이지 "다음 루프의 프로그램"이 아님: 결정 테이블이 §6.1 좋은 목표의 요소(성공 기준·변경 금지 제약·검증·기각 대안)를 인코딩하지 않음. feature-draft orchestrator가 "기각한 대안"을 digest로 수집하는데 discussion 요약엔 해당 섹션 부재. (갭 2) 결정 "근거"의 evidence/confidence 미구분 — 미검증 가정 위 결정이 확정 사실처럼 후속 루프로 넘어가는 것이 §10.3 context drift의 씨앗.
2. **표현 방식**: loop-eng 용어를 표면에 도입할지 vs 구조만 체화할지 — 기존 규범 원칙(스킬은 부합만, 참조 안 함)과의 일관성 문제.
3. **형식 비대 리스크 (비판적 검토)**: 새 필드를 모든 요약에 강제하면 탐색적 토론에서 "해당 없음" filler 생산 — §6.4가 경계하는 가짜 신호이자 출력 다이어트 방향과 역행.
4. **결정 재방문 규칙의 thrashing 리스크**: 전제 모순 시 결정을 재방문하는 규칙 자체가 §10.1 thrashing을 만들 수 있어 중단 조건 필요.

## 리서치 결과 요약 (Research Findings)

- **정렬 진단표** (loop-eng 5조건 × 진행/요약): 명확한 목표·작은 변경·중단 조건·실패 모드 대응은 진행 쪽에서 모두 체화 확인. 관찰 신호(§6.4)만 양쪽에서 미흡 — 진행은 주장/가정 검증 여부 구분 없음, 요약은 근거의 evidence/confidence 구분 없음.
- **audit 교차 확인**: `docs/audits/discussion_conformance.md` CLEAR — 본 토론의 갭은 규범 위반이 아니라 부합-이상의 철학 체화 영역임을 확인.

## 토론 흐름 (Discussion Flow)

Round 1: 문제의식 확인 → 철학적 정렬 자체가 목적 (구체 불만 아님)
Round 2: 표현 방식 → 구조만 체화 (규범 원칙 유지, SDD 어휘)
Round 3: 접근 선택 (3안 + 권장) → 방안 1(산출물 정렬) + 결정 재방문 규칙
Round 4: 비판적 검토 (형식 비대/filler 리스크) → 조건부 적용 채택
Round 5: 재방문 규칙 사양 승인 + 수렴 확인 → 정리
