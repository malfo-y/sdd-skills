# Discussion Question Guide (Codex Plan Mode)

`discussion` 스킬에서 사용하는 질문 전략 및 템플릿 가이드.

## Plan Mode 전제

- 사용자 질문은 Plan mode에서만 `request_user_input`으로 수행한다.
- 옵션은 2~3개로 구성한다.
- 권장 형식은 `실질 선택지 2개 + 토론 종료/정리 1개`이다.
- 토론 종료 후 저장이 필요하면 Plan mode에서 저장 의사를 확인하고, 실제 저장은 Plan mode 종료 후 핸드오프로 실행한다.

## 토론 단계별 질문 전략

### Phase 1: 탐색 (Exploration) - Round 1~3

목적: 토픽의 범위와 목표를 명확히 한다.

| 질문 유형 | 템플릿 | 사용 시점 |
|-----------|--------|----------|
| 동기 확인 | "이 [토픽]을 논의하는 핵심 이유는 무엇인가요?" | Round 1 |
| 목표 정의 | "이번 토론이 끝났을 때 어떤 결론이 나오면 좋을까요?" | Round 1~2 |
| 제약 확인 | "꼭 지켜야 할 제약(시간/기술/리스크)이 있나요?" | Round 2~3 |

### Phase 2: 분석 (Analysis) - Round 4~6

목적: 대안을 비교하고 트레이드오프를 분석한다.

| 질문 유형 | 템플릿 | 사용 시점 |
|-----------|--------|----------|
| 대안 비교 | "A와 B 중 [기준]에서 무엇이 더 적합한가요?" | Round 4~5 |
| 트레이드오프 | "[장점]을 얻기 위해 [단점]을 감수할 수 있나요?" | Round 4~5 |
| 우선순위 | "결정 기준 중 가장 중요한 것은 무엇인가요?" | Round 5~6 |

### Phase 3: 수렴 (Convergence) - Round 7+

목적: 결론을 확정하고 실행 항목을 명시한다.

| 질문 유형 | 템플릿 | 사용 시점 |
|-----------|--------|----------|
| 결론 확인 | "지금까지 논의를 기준으로 [결론]으로 확정할까요?" | Round 7+ |
| 위험 점검 | "이 결정의 최대 리스크는 무엇인가요?" | Round 7+ |
| 실행 정리 | "다음 단계에서 먼저 실행할 항목은 무엇인가요?" | Round 8+ |

## 옵션 설계 원칙

1. **상호 배타성**: 옵션 간 의미가 겹치지 않아야 한다.
2. **포괄성**: 주요 대안을 충분히 커버해야 한다.
3. **명확성**: 사용자가 선택 의미를 즉시 이해할 수 있어야 한다.
4. **종료 가능성**: 종료 옵션을 통해 언제든 요약으로 전환 가능해야 한다.

## 종료 시 저장 질문 템플릿

토론 요약 출력 후 아래 질문을 사용한다.

```text
질문: "토론 요약을 파일로 저장할까요?"
옵션:
1. "저장하지 않음"
2. "기본 경로로 저장 준비"
3. "경로 지정 후 저장 준비"
```

저장 선택 시 출력해야 할 정보:
- `target_path`
- `overwrite_policy`
- Plan mode 종료 후 실행할 저장 요청 문장

## 리서치 트리거

### 로컬 우선 탐색

- 코드 구조/패턴/제약 확인이 필요할 때
- 기존 구현 근거가 토론 결론에 직접 영향을 줄 때

### 선택적 웹 리서치

- 기술 비교가 최신 버전/정책에 좌우될 때
- 코드베이스 근거만으로 결론이 어려울 때
- 변동 가능성이 높은 사실 검증이 필요할 때

## 상태 추적 예시

```text
## Discussion State
- Topic: [토픽]
- Current Phase: Exploration / Analysis / Convergence
- Round: N / MAX(10)

### Key Points
1. [논점]: [요약] (Round N)

### Decisions
1. [결정]: [근거] (Round N)

### Open Questions
1. [질문]: [맥락]

### Action Items
1. [항목]: [우선순위]

### Research Log
1. [주제]: [핵심 결과] (Round N, local/web)
```
