# 기능 기술 보고서: 활동 로그 내보내기

**생성일**: 2026-03-12
**입력 소스**: spec (코드 근거 없음)
**대상 기능**: 활동 로그 내보내기
**신뢰도**: Low

## 1. 배경 및 동기

활동 로그 내보내기 기능은 관리자가 워크스페이스 내 사용자 활동 이력을 CSV 또는 JSON 형식으로 다운로드할 수 있게 하는 기능이다.

이 기능이 해결하는 문제:
- 감사(audit) 목적의 활동 기록을 외부로 추출하여 보관 및 제출 가능
- 외부 분석 도구(BI, SIEM 등)와 연계하여 활동 패턴 분석 가능
- 규정 준수를 위한 데이터 보존 요구사항 충족

실시간 스트리밍 대신 파일 기반 내보내기를 택한 이유는, 대부분의 감사/분석 시나리오가 과거 데이터를 일괄 처리하는 형태이며, 파일 기반이 기존 워크플로우와의 호환성이 높기 때문으로 추정된다 (스펙에서 명시적 이유는 확인되지 않음).

## 2. 핵심 설계

핵심 설계는 **필터 기반 조회 + 비동기 파일 생성 + 임시 다운로드 URL**로 추정된다.

- 내보내기 요청 시 기간 필터(필수, 최대 90일)와 출력 형식(CSV/JSON)을 지정한다.
- 대용량 데이터는 비동기 처리한다 (스펙에 "백그라운드 처리" 언급 있으나 임계값 미명시).
- 생성된 파일은 임시 URL로 다운로드하며, 보관 기간은 24시간으로 가정한다.
- 접근 권한은 관리자로 제한한다 (PII 포함 가능성).

관련 코드:
- 관련 구현 코드가 확인되지 않음. 구현 시 아래 코드 레퍼런스를 갱신할 것을 권장한다.

## 3. 사용 시나리오 가이드

### 시나리오 1: 정상 내보내기 (CSV, 소규모)

**전제 조건**: 관리자가 로그인되어 있고, 내보내기 대상 기간의 활동 로그가 존재.

**입력**:
```json
POST /workspaces/ws_001/activity-logs/export
{
  "start_date": "2026-02-12",
  "end_date": "2026-03-12",
  "format": "csv"
}
```

**처리 흐름**:
1. 요청자의 관리자 권한 확인
2. 기간 검증: `end_date - start_date ≤ 90일` → 통과
3. 활동 로그 조회 (데이터 소규모 → 동기 처리)
4. CSV 형식으로 변환
5. 임시 파일 생성 및 다운로드 URL 발급

**기대 결과**:
```json
{
  "export_id": "exp_001",
  "status": "completed",
  "download_url": "/downloads/exp_001.csv",
  "expires_at": "2026-03-13T10:30:00Z",
  "row_count": 1250,
  "format": "csv"
}
```

### 시나리오 2: 대용량 비동기 내보내기

**전제 조건**: 내보내기 대상 데이터가 임계값을 초과하여 비동기 처리가 필요.

**입력**:
```json
POST /workspaces/ws_001/activity-logs/export
{
  "start_date": "2025-12-12",
  "end_date": "2026-03-12",
  "format": "json"
}
```

**처리 흐름**:
1. 권한 확인 및 기간 검증 → 통과 (90일)
2. 예상 데이터량이 임계값 초과 → 비동기 작업 생성
3. 즉시 작업 ID 반환
4. 백그라운드에서 파일 생성
5. 완료 시 알림 (이메일 또는 인앱 알림, 방식 미확정)

**기대 결과** (즉시 응답):
```json
{
  "export_id": "exp_002",
  "status": "processing",
  "estimated_completion": "2026-03-12T10:35:00Z"
}
```

**완료 후 조회 결과**:
```json
{
  "export_id": "exp_002",
  "status": "completed",
  "download_url": "/downloads/exp_002.json",
  "expires_at": "2026-03-13T10:35:00Z",
  "row_count": 45000,
  "format": "json"
}
```

### 시나리오 3: 90일 초과 기간 요청

**전제 조건**: 요청 기간이 90일을 초과.

**입력**:
```json
POST /workspaces/ws_001/activity-logs/export
{
  "start_date": "2025-01-01",
  "end_date": "2026-03-12",
  "format": "csv"
}
```

**기대 결과**:
```json
{
  "error": "PERIOD_EXCEEDED",
  "message": "내보내기 기간은 최대 90일입니다. 기간을 조정해 주세요.",
  "max_days": 90,
  "requested_days": 436
}
```
HTTP 상태: 422 Unprocessable Entity

### 시나리오 4: 비관리자 접근

**전제 조건**: 일반 사용자(관리자 권한 없음)가 내보내기 시도.

**기대 결과**:
```json
{
  "error": "FORBIDDEN",
  "message": "활동 로그 내보내기는 관리자만 사용할 수 있습니다."
}
```
HTTP 상태: 403 Forbidden

### 시나리오 5: 빈 결과 내보내기

**전제 조건**: 지정 기간 내 활동 로그가 없음.

**기대 결과** (추정):
```json
{
  "export_id": "exp_003",
  "status": "completed",
  "download_url": "/downloads/exp_003.csv",
  "row_count": 0,
  "format": "csv"
}
```
빈 파일이지만 헤더 행은 포함될 것으로 가정 (CSV 기준).

## 4. API 레퍼런스

### POST /workspaces/:workspace_id/activity-logs/export

활동 로그 내보내기를 요청한다.

**URL 파라미터**:

| 이름 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `workspace_id` | string | 필수 | 대상 워크스페이스 ID |

**요청 본문**:

| 이름 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `start_date` | string (YYYY-MM-DD) | 필수 | 조회 시작일 |
| `end_date` | string (YYYY-MM-DD) | 필수 | 조회 종료일 (`end_date - start_date ≤ 90일`) |
| `format` | string | 선택 | `"csv"` (기본값, 추정) 또는 `"json"` |

**성공 응답** (200 OK 또는 202 Accepted):

| 필드 | 타입 | 설명 |
|------|------|------|
| `export_id` | string | 내보내기 작업 식별자 |
| `status` | string | `"completed"` 또는 `"processing"` |
| `download_url` | string | 다운로드 URL (완료 시) |
| `expires_at` | string (ISO 8601) | 다운로드 URL 만료 시각 (24시간 후 가정) |
| `row_count` | number | 내보내기된 행 수 (완료 시) |
| `format` | string | 출력 형식 |
| `estimated_completion` | string (ISO 8601) | 예상 완료 시각 (비동기 시) |

**에러 응답**:

| HTTP 상태 | 에러 코드 | 상황 |
|-----------|----------|------|
| 403 | `FORBIDDEN` | 관리자 권한 없음 |
| 404 | `WORKSPACE_NOT_FOUND` | 존재하지 않는 워크스페이스 |
| 422 | `PERIOD_EXCEEDED` | 90일 초과 기간 |
| 422 | `INVALID_DATE_RANGE` | 잘못된 날짜 범위 (시작일 > 종료일 등) |
| 422 | `INVALID_FORMAT` | 지원하지 않는 형식 |

---

### GET /workspaces/:workspace_id/activity-logs/exports/:export_id

내보내기 작업 상태를 조회한다 (비동기 처리 시 폴링용, 추정).

**성공 응답** (200 OK): POST 응답과 동일한 구조

**호출 예시**:

```bash
# 내보내기 요청
curl -X POST /api/v1/workspaces/ws_001/activity-logs/export \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <admin_token>" \
  -d '{
    "start_date": "2026-02-12",
    "end_date": "2026-03-12",
    "format": "csv"
  }'

# 비동기 작업 상태 확인
curl /api/v1/workspaces/ws_001/activity-logs/exports/exp_002 \
  -H "Authorization: Bearer <admin_token>"
```

## 5. 구현 가이드

### 핵심 규칙

1. 내보내기 요청 시 기간 필터는 필수이며, 최대 90일로 제한한다 (스펙 기준).
2. 출력 형식은 CSV와 JSON을 지원한다 (기본값은 CSV로 추정).
3. 대용량 데이터는 비동기 처리한다 (임계값 미확정).
4. PII 포함 가능성이 있으므로 접근 권한을 관리자로 제한한다.
5. 완료된 파일의 보관 기간은 24시간으로 가정한다.

### 체크리스트

**구현 전**:
- [ ] 스펙에서 활동 로그 데이터 모델(이벤트 타입, 필드)을 확인했다.
- [ ] 비동기 처리 필요 여부와 임계값을 결정했다.
- [ ] 기존 활동 로그 저장소(DB 테이블, 인덱스)를 확인했다.
- [ ] 파일 임시 저장 위치와 보관 정책을 결정했다.

**구현 중**:
- [ ] 기간 필터 필수 검증과 90일 상한 검증이 있다.
- [ ] CSV/JSON 출력 형식 분기가 있다.
- [ ] 관리자 권한 확인이 요청 처리 전에 실행된다.
- [ ] 대용량 판단 기준과 비동기 전환 로직이 있다.

**완료/리뷰 전**:
- [ ] 정상 내보내기 플로우 테스트가 있다.
- [ ] 90일 초과 기간 요청 거부 테스트가 있다.
- [ ] 비관리자 접근 차단 테스트가 있다.
- [ ] CSV와 JSON 각각에 대한 출력 검증 테스트가 있다.

### 안티패턴

- 기간 제한 없이 전체 이력을 한 번에 조회하여 메모리를 과도하게 사용하는 방식 — OOM 위험
- 내보내기 파일을 영구 저장하여 PII가 포함된 파일이 무기한 남는 방식 — 개인정보 보호 위반
- 일반 사용자도 타인의 활동 로그를 내보낼 수 있는 방식 — 권한 우회

## 부록

### 관련 스펙 레퍼런스

- `_sdd/spec/admin.md` → `기능 > 활동 로그 내보내기`
- `_sdd/spec/admin.md` → `데이터 모델 > 활동 이벤트`

### 관련 코드 레퍼런스

- 관련 구현 코드가 확인되지 않음. 구현 시 파일 경로와 심볼을 이 부록에 추가할 것을 권장한다.

### 가정 및 미확정 사항

- 내보내기 파일 보관 기간은 스펙에서 확인되지 않아 24시간으로 가정했다.
- 비동기 처리 임계값(행 수 또는 기간 기준)은 스펙에 구체적 수치가 없어 구현 시 결정이 필요하다.
- 출력 형식의 기본값은 스펙에서 명시되지 않아 CSV를 기본값으로 추정했다.
- 활동 로그 테이블 구조나 인덱스 현황은 코드 확인이 필요하다.
- 비동기 내보내기 완료 알림 방식(이메일/인앱)은 스펙에서 확인되지 않았다.
- GET 상태 조회 API는 스펙에서 명시되지 않아 존재를 추정한 것이다.
