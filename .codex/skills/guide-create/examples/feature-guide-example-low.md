# 기능 가이드: 활동 로그 내보내기

**생성일**: 2026-03-12
**입력 소스**: spec (코드 근거 없음)
**대상 기능**: 활동 로그 내보내기
**신뢰도**: Low

## 설명

활동 로그 내보내기 기능은 관리자가 워크스페이스 내 사용자 활동 이력을 CSV 또는 JSON 형식으로 다운로드할 수 있게 하는 기능이다. 이 기능의 사용자 가치는 감사(audit) 목적의 기록 확보와 외부 분석 도구 연계에 있다.

이 가이드는 "내보내기 요청 → 필터 적용 → 파일 생성 → 다운로드"까지를 다룬다. 활동 로그 수집/저장은 이미 존재한다고 가정하며, 실시간 스트리밍 내보내기는 범위에 포함하지 않는다. 현재 관련 구현 코드가 확인되지 않아 스펙 근거 중심으로 작성되었다.

## 규칙

1. 내보내기 요청 시 기간 필터는 필수이며, 최대 조회 범위는 90일로 제한한다 (스펙 기준).
2. 출력 형식은 `csv`와 `json` 두 가지를 지원한다 (스펙 기준, 기본값은 `csv`로 추정).
3. 대량 데이터 내보내기는 비동기 처리가 권장된다. 스펙에 "대용량 시 백그라운드 처리" 언급이 있으나 구체적 임계값은 명시되어 있지 않다.
4. 내보내기 파일에는 개인 식별 정보(PII)가 포함될 수 있으므로 접근 권한을 관리자로 제한한다 (스펙 기준).
5. 완료된 내보내기 파일의 보관 기간은 스펙에서 확인되지 않아 24시간 임시 보관으로 가정한다.

## 체크리스트

### 구현 전

- [ ] 스펙에서 활동 로그 데이터 모델(이벤트 타입, 필드)을 확인했다.
- [ ] 내보내기 API 엔드포인트 설계를 스펙에서 확인했다.
- [ ] 비동기 처리 필요 여부와 임계값을 결정했다.
- [ ] 기존 활동 로그 저장소(DB 테이블, 인덱스)를 확인했다.

### 구현 중

- [ ] 기간 필터 필수 검증과 90일 상한 검증이 있다.
- [ ] CSV/JSON 출력 형식 분기가 있다.
- [ ] 관리자 권한 확인이 요청 처리 전에 실행된다.
- [ ] 대용량 판단 기준과 비동기 전환 로직이 결정되었다.

### 완료/리뷰 전

- [ ] 정상 내보내기 플로우 테스트가 있다.
- [ ] 90일 초과 기간 요청 거부 테스트가 있다.
- [ ] 비관리자 접근 차단 테스트가 있다.
- [ ] CSV와 JSON 형식 각각에 대한 출력 검증 테스트가 있다.

## 예시

### 좋은 예시

관리자가 "최근 30일 활동 로그를 CSV로 내보내기"를 요청하면, 서버는 먼저 요청자의 관리자 권한을 확인한다. 기간이 90일 이하인지 검증한 뒤 활동 로그를 조회하고, CSV 형식으로 변환하여 임시 파일을 생성한다. 생성 완료 후 다운로드 URL을 반환하고, 해당 파일은 24시간 뒤 자동 삭제된다.

### 안티패턴

- 기간 제한 없이 전체 이력을 한 번에 조회하여 메모리를 과도하게 사용하는 방식
- 내보내기 파일을 영구 저장하여 PII가 포함된 파일이 무기한 남는 방식
- 일반 사용자도 타인의 활동 로그를 내보낼 수 있는 방식

### 관련 코드 레퍼런스

- 관련 구현 코드가 확인되지 않음. 구현 시 파일 경로와 심볼을 이 부록에 추가할 것을 권장한다.

### 가정 및 미확정 사항

- 내보내기 파일 보관 기간은 스펙에서 확인되지 않아 24시간으로 가정했다.
- 비동기 처리 임계값(행 수 또는 기간 기준)은 스펙에 구체적 수치가 없어 구현 시 결정이 필요하다.
- 출력 형식의 기본값은 스펙에서 명시되지 않아 CSV를 기본값으로 추정했다.
- 활동 로그 테이블 구조나 인덱스 현황은 코드 확인이 필요하다.
